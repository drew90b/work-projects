# Weekly Review Template

**Last Updated:** 2026-03-19

> Run this every week — Friday afternoon or weekend. GTD only works if the system is current.
> Copy this checklist into a new file at `weekly-checks/YYYY-WW.md` for each review.

---

## Step 1 — Get Clear (Capture & Clarify)

- [ ] Process inbox to zero (`inbox/inbox.md`)
- [ ] Review notes from the week (Telegram, paper, voice memos)
- [ ] Review daily notes folder (`daily-notes/`)
- [ ] Capture any open loops still in your head

## Step 2 — Get Current (Review & Update)

- [ ] Review **Next Actions** — mark completed, add new ones (`next-actions/next-actions.md`)
- [ ] Review **Waiting For** — follow up on anything overdue (`next-actions/waiting-for.md`)
- [ ] Review **Projects List** — every project has a next action? (`projects/projects-list.md`)
- [ ] Review **Upcoming Deadlines** in `projects/reliability.md` and `projects/ms-project.md`
- [ ] Update **README.md** — top 5 priorities for next week

## Step 3 — Get Creative (Someday/Maybe)

- [ ] Review **Someday/Maybe** — anything to promote to an active project?
  (`reference/someday-maybe.md`)
- [ ] Any new ideas or opportunities to capture?
